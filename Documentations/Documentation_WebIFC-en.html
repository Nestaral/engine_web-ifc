Description

engine_web-ifc is a fork/copy of the open-source web-ifc library by That Open Company. It provides a JavaScript/WebAssembly library for reading and writing IFC (Industry Foundation Classes) files at native speeds, targeting both browser and Node.js environments.

This is the core IFC parsing engine used by the BIM viewer projects in this workspace. The C++ core is compiled to WebAssembly via Emscripten, while a TypeScript API wrapper provides a developer-friendly interface with full type safety.

Key Features

Native-Speed IFC Parsing

C++ parser compiled to WebAssembly for near-native performance when reading and writing IFC files in the browser or Node.js, handling models with millions of elements.

Multi-Schema Support

Supports IFC2X3, IFC4, and IFC4X3 schemas. Automatically detects the schema from the FILE_SCHEMA header and applies the correct type constructors and inheritance definitions.

Full Read/Write Capability

Read existing IFC files, create new models programmatically, modify elements, write lines, delete entities, and save back to IFC format. Complete CRUD operations on IFC data.

3D Geometry Engine

Built-in geometry extraction: mesh streaming, flat mesh generation, extrusions, sweeps, revolutions, boolean operations, profile sections, arcs, clothoids, parabolas, and alignment data.

Streaming API

Stream meshes one at a time via callbacks (StreamAllMeshes, StreamMeshes, StreamAllMeshesWithTypes) to handle large models without loading all geometry into memory at once.

Multi-Threading Support

Optional multi-threaded WASM build (web-ifc-mt.wasm) for cross-origin isolated environments, with automatic fallback to single-threaded mode.

Cross-Platform Distribution

Three build targets: ESM for browsers, CommonJS for Node.js, and IIFE for script-tag inclusion. Separate WASM files for browser, Node.js, and multi-threaded contexts.

Installation & Build

Prerequisites

  • Node.js v16+ and npm v7+
  • Emscripten SDK v3.1.44+ (for WASM compilation)
  • CMake v3.18+ (for C++ build system)

Install Dependencies

cd Sources
npm install

Full Release Build (WASM + JS API)

npm run setup-env          # Set up Emscripten environment variables
npm run build-release      # Build WASM + JS API (output in dist/)

Debug Build

npm run build-debug        # Build with debug symbols

Development Viewer

npm run dev                # Launch local viewer on port 8080

Run Tests

npm test                   # Jest test suite
npm run benchmark          # Performance benchmarks
npm run regression         # Regression test suite

Docker Build

npm run docker-build       # Build Docker image
npm run docker-run         # Run container and extract compiled files

Schema Generation

npm run gen-schema         # Regenerate IFC schema TypeScript from .exp files

Usage Guide

Browser (ESM)

import { IfcAPI } from 'web-ifc';

const ifcApi = new IfcAPI();
ifcApi.SetWasmPath('./');
await ifcApi.Init();

// Load an IFC file
const response = await fetch('model.ifc');
const data = new Uint8Array(await response.arrayBuffer());
const modelID = ifcApi.OpenModel(data);

// Read all elements of a specific type
const lines = ifcApi.GetLineIDsWithType(modelID, IFCDOOR);
for (let i = 0; i < lines.size(); i++) {
  const door = ifcApi.GetLine(modelID, lines.get(i));
  console.log(door.Name.value, door.GlobalId.value);
}

// Stream all meshes for 3D rendering
ifcApi.StreamAllMeshes(modelID, (mesh, index, total) => {
  // Process mesh geometry
});

ifcApi.CloseModel(modelID);

Node.js (CommonJS)

const { IfcAPI } = require('web-ifc');
const fs = require('fs');

const ifcApi = new IfcAPI();
await ifcApi.Init();

const data = fs.readFileSync('model.ifc');
const modelID = ifcApi.OpenModel(new Uint8Array(data));

// Get model schema version
const schema = ifcApi.GetModelSchema(modelID);
console.log(`Schema: ${schema}`); // "IFC4"

// Save modified model
const savedData = ifcApi.SaveModel(modelID);
fs.writeFileSync('output.ifc', savedData);

ifcApi.CloseModel(modelID);

Create a New Model

const modelID = ifcApi.CreateModel({
  schema: 'IFC4',
  name: 'MyProject.ifc',
  description: ['ViewDefinition [CoordinationView]'],
  authors: ['Mathis Centauri'],
  organizations: ['My Company']
});

// Write entities to the model
const wall = ifcApi.CreateIfcEntity(modelID, IFCWALL, ...args);
ifcApi.WriteLine(modelID, wall);

const data = ifcApi.SaveModel(modelID);

Geometry Engine

The C++ geometry engine provides factory methods for creating and computing various geometric primitives:

FactoryDescription
CreateExtrusion()Linear extrusion of a profile along a direction
CreateSweep()Profile swept along a directrix curve
CreateCircularSweep()Profile swept along a circular arc
CreateRevolution()Profile revolved around an axis
CreateCylindricalRevolution()Cylindrical revolution geometry
CreateParabola()Parabolic curve geometry
CreateClothoid()Clothoid (Euler spiral) curve
CreateArc()Elliptical arc geometry
CreateBooleanOperator()CSG boolean operations (union, subtraction, intersection)
CreateAABB()Axis-aligned bounding box
CreateProfile()2D profile section (I-beam, T-section, etc.)
